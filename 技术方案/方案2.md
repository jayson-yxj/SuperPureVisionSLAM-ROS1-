# #1️⃣ ORB-SLAM3（单目） → 稳定实时的位姿（Orin 上 40–60 FPS）

这是你 **单目 SLAM 的核心**。

特点：

* ROS1 支持非常成熟
* Orin 上单目能跑很快
* 提供高质量位姿给所有后端模块

开源：ORB-SLAM3（官方）

---

# #2️⃣ FastDepth（或 MiDaS small） → 实时单目深度（Orin 上 30–60 FPS）

单目稠密深度是构建 Occupancy/TSDF 的基础。

**推荐：FastDepth（MobileNet）**

* Orin 上 real-time（60FPS 没问题）
* 输出 dense depth

备选：MiDaS small（更准确，但略慢）

你可以直接用 **TensorRT 加速**。

---

# #3️⃣ Voxblox（ROS1 完全兼容） → TSDF/占据融合

这是你构建稠密地图的核心：

## Voxblox 的优点

* 专为机器人实时建图设计
* 支持 TSDF、ESDF（可用于导航）
* 输入：**位姿 + 深度图**
* 输出：**3D TSDF + 可转 2D 占据地图**

Orin 上可以实时。

## Voxblox 输出：

* 3D TSDF 网格（可保存）
* ESDF（可做避障）
* 2D 占据地图（move_base 直接用）

你把 TSDF 投影成 2D occupancy grid → 交给 ROS1 导航即可。

---

# #4️⃣ 2D 导航（ROS1 move_base）

通过 Voxblox 生成的 2D occupancy map：

* 用 **map_server** 发布 2D 地图
* costmap_2d 直接使用
* 使用 DWA/TEB 实现本体导航

这是最稳定、工程可落地的方式。

---

# #5️⃣ 如果你还需要 3D 和语义（可选模块）

Orin 的算力足以支持更高级的 3D 占据、语义网络。

## 🟣 3D Occupancy（深度学习版）

推荐以下真正可跑的模型：

### **1）FastOcc**

* 多视几何 + 单目深度输入
* 实时 3D Occupancy
* Orin 上 TensorRT 推理可达 15–20 FPS

### **2）MonoScene（语义+占据）**

* CVPR 2022
* 单目 → 3D Semantic Occupancy
* Orin 上 5–10 FPS（取决于输入分辨率）

### 输出：

* 3D 语义体素
* Occupancy（用于 3D 探索、路径规划）

你可以将其作为 **Voxblox 的高阶语义版本**。
